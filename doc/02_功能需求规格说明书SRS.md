# 功能需求规格说明书 SRS

> 项目：KnowPilot AI Agent 本地知识库工具  
> 用途：指导产品、前端、后端、算法、测试和运维团队协同研发  
> 版本：v0.1  
> 日期：2026-05-15


## 1. 文档目的

本文档面向研发、测试和产品团队，描述 KnowPilot MVP 的详细功能需求、业务规则、边界条件和异常处理。

## 2. 角色定义

| 角色 | 权限 |
|---|---|
| 个人用户 | 导入、编辑、搜索、标注、问答、管理实体和图谱 |
| 管理员 | 管理模型配置、存储配置、索引重建和备份恢复 |
| 系统任务 | 执行解析、OCR、向量化、实体抽取、图谱同步 |

MVP 默认单用户，权限模型先保留字段，不做复杂角色管理。

## 3. 导入中心需求

### 3.1 文件导入

系统应支持以下文件：

| 类型 | 后缀 | MVP 处理方式 |
|---|---|---|
| PDF | pdf | 文本提取，必要时 OCR |
| Word | docx | 提取标题、段落、表格 |
| Excel | xlsx/csv | 提取 Sheet、表格、字段 |
| 图片 | png/jpg/jpeg/webp | OCR + 图像描述可选 |
| Markdown | md | 原样保存并结构化 |
| 文本 | txt | 原样保存并切分 |

业务规则：

1. 上传文件后先保存原始文件。
2. 计算 sha256，重复文件应提示。
3. 创建导入任务，任务状态可查询。
4. 解析成功后生成文档记录和版本记录。
5. 解析失败时保存错误原因，允许重试。
6. 文件大小 MVP 默认限制 200MB，可配置。

### 3.2 URL 导入

需求：

1. 用户输入 URL 后，系统抓取网页标题、正文、发布时间、作者、正文图片链接。
2. 抓取失败时保存 URL 记录并提示原因。
3. 支持保存网页快照 HTML。
4. 支持将网页加入深度研究任务。

### 3.3 导入任务状态

| 状态 | 说明 |
|---|---|
| pending | 等待处理 |
| reading | 读取文件 |
| parsing | 解析正文 |
| ocr_running | OCR 中 |
| chunking | 文档切分 |
| embedding | 向量化 |
| extracting_entities | 实体识别 |
| extracting_relations | 关系抽取 |
| indexing | 写入索引 |
| completed | 完成 |
| failed | 失败 |

## 4. 文档管理需求

### 4.1 文档列表

列表字段：

- 标题
- 来源类型
- 分类
- 标签
- 摘要
- 处理状态
- 更新时间
- 实体数量
- 关系数量

筛选条件：

- 分类
- 标签
- 来源类型
- 时间范围
- 处理状态
- 是否有标注
- 是否有低置信关系

### 4.2 分类

规则：

1. 分类最多 3 级。
2. 文档最多有一个主分类。
3. 文档可有多个标签。
4. 自动分类结果必须允许用户修改。
5. 用户修改分类后，应记录为后续分类参考。

### 4.3 文档编辑

MVP 使用 Markdown 编辑器。

需求：

1. 用户可以编辑标题和正文。
2. 保存时创建新版本。
3. 保存后触发重新切分和索引任务。
4. 是否重新实体识别由用户选择，默认提醒。
5. 可查看版本历史。
6. 可回滚到历史版本。

## 5. 标注备注需求

标注类型：

| 类型 | 说明 |
|---|---|
| highlight | 高亮 |
| underline | 下划线 |
| note | 备注 |
| excerpt | 摘录卡片 |
| question | 待研究问题 |
| todo | 待办 |

业务规则：

1. 标注必须绑定 doc_id 和 version_id。
2. 标注尽量绑定 chunk_id。
3. 标注内容应保存 selected_text，避免版本变更后无法定位。
4. 删除文档时，标注软删除。
5. 摘录卡片可作为独立知识项参与搜索。

## 6. 搜索和问答需求

### 6.1 搜索类型

1. 关键词搜索。
2. 语义搜索。
3. 图谱搜索。
4. 标注搜索。
5. 网络搜索。

### 6.2 RAG 问答规则

1. 回答必须基于检索到的上下文。
2. 回答必须带引用来源。
3. 没有足够证据时，应明确提示。
4. 不允许伪造引用。
5. 股票、政策、法律、行情类问题需要提示数据日期。

## 7. 实体识别需求

MVP 内置实体类型：

- 人物
- 组织/公司
- 产品
- 技术
- 项目
- 地点
- 时间
- 事件
- 指标
- IP/端口
- 配置项
- 文件路径
- 错误信息
- 行业
- 产业链
- 产业链环节
- 股票/证券
- 财务指标
- 估值指标
- 供应商
- 客户
- 竞争对手
- 政策法规
- 风险因素

业务规则：

1. 实体识别结果必须包含 evidence。
2. 置信度低于阈值的实体进入待确认。
3. 用户可以合并、删除、编辑实体。
4. 股票实体需要识别 ticker、exchange、company_name、industry、sector。
5. 产业链实体需要识别 upstream、midstream、downstream、core_companies。
6. 自动发现实体类型必须由用户确认后才生效。

## 8. 关系抽取需求

内置关系类型：

- 属于
- 依赖
- 供应给
- 客户是
- 竞争
- 上游
- 下游
- 影响
- 导致
- 解决
- 部署于
- 连接到
- 报错于
- 发布
- 投资/持股

业务规则：

1. 关系必须绑定 source_entity、target_entity、relation_type。
2. 关系必须绑定 evidence_text。
3. 不确定关系不入正式图谱，进入待确认。
4. 用户确认后的关系标记 verified=true。
5. 删除实体时，相关关系应软删除或标记失效。

## 9. 图谱需求

MVP 图谱能力：

1. 展示实体节点和关系边。
2. 支持节点类型筛选。
3. 支持关系类型筛选。
4. 支持节点一跳展开。
5. 支持节点详情面板。
6. 支持查看关系证据。
7. 支持编辑节点和关系。
8. 默认最多加载 300 个节点。

## 10. 深度研究需求

MVP 简化版：

1. 用户输入研究问题。
2. 系统生成研究计划。
3. 搜索本地知识库。
4. 可选网络搜索。
5. 生成报告。
6. 报告可以作为文档导入知识库。
7. 报告中的实体和关系可二次抽取。

## 11. NotebookLM 需求

MVP 只做资料包导出：

1. 用户选择文档集合或分类。
2. 系统导出 ZIP。
3. ZIP 包含 README、文档正文、实体表、时间线、建议问题。
4. 用户手动上传到 NotebookLM。
5. 系统记录导出历史。

## 12. 非功能需求

| 类型 | 要求 |
|---|---|
| 性能 | 普通搜索 2 秒内返回，图谱一跳 2 秒内返回 |
| 可用性 | 文件解析失败可重试，任务状态可追踪 |
| 隐私 | 敏感文档默认不调用云模型 |
| 可追溯 | AI 结果必须绑定来源 |
| 可维护 | 所有模型输出必须走 Schema 校验 |
| 可恢复 | 支持备份和恢复本地数据 |
