# 用户故事与验收标准

> 项目：KnowPilot AI Agent 本地知识库工具  
> 用途：指导产品、前端、后端、算法、测试和运维团队协同研发  
> 版本：v0.1  
> 日期：2026-05-15


## 1. 编写原则

每条用户故事都应满足：

```text
作为某类用户，我希望完成某个动作，以便获得某个价值。
```

每条用户故事必须配套验收标准，验收标准应可测试、可判断、可关闭。

---

## 2. Epic：资料导入

### US-001 上传 PDF 文档

作为用户，我希望上传 PDF 文档后，系统自动解析正文并加入知识库，以便后续搜索和问答。

验收标准：

- 用户可以选择或拖拽 PDF 文件。
- 系统创建导入任务。
- 系统保存原始 PDF。
- 系统提取 PDF 文本。
- 若 PDF 无文本层，系统尝试 OCR。
- 解析结果生成 Markdown 文档。
- 文档出现在知识库列表中。
- 文档可以被全文搜索命中。
- 失败时显示错误原因和重试按钮。

### US-002 上传 Excel 文件

作为用户，我希望上传 Excel 后，系统能识别 Sheet 和表格内容，以便后续对表格资料进行问答和实体抽取。

验收标准：

- 支持 xlsx 和 csv。
- 能列出 Sheet 名称。
- 能将表格转成 Markdown 表格。
- 能识别表头。
- 能提取公司、股票代码、财务指标等实体。
- 表格内容可被搜索。

### US-003 URL 导入网页

作为用户，我希望粘贴 URL 后自动保存网页正文，以便把网络资料沉淀到本地知识库。

验收标准：

- 用户可以输入 URL。
- 系统抓取网页标题和正文。
- 系统保存来源 URL。
- 系统生成摘要。
- 系统识别实体。
- 抓取失败时保存失败原因。

---

## 3. Epic：文档管理

### US-004 查看文档列表

作为用户，我希望按分类、标签、类型查看文档，以便快速找到资料。

验收标准：

- 文档列表展示标题、摘要、标签、分类、更新时间。
- 支持按分类筛选。
- 支持按标签筛选。
- 支持按来源类型筛选。
- 支持关键词搜索。

### US-005 编辑解析后的文档

作为用户，我希望修改解析后的文档内容并保存，以便修正 OCR 或解析错误。

验收标准：

- 可以进入文档编辑模式。
- 可以修改标题和正文。
- 保存后生成新版本。
- 可以查看版本历史。
- 可以回滚历史版本。
- 保存后触发重新索引。

---

## 4. Epic：标注备注

### US-006 高亮文档片段

作为用户，我希望在阅读时高亮重点内容，以便后续复习和检索。

验收标准：

- 用户可选中文本。
- 可选择高亮颜色。
- 高亮内容保存到 annotation 表。
- 重新打开文档后高亮仍存在。
- 高亮内容可被搜索。

### US-007 添加备注

作为用户，我希望对文档片段添加备注，以便记录自己的理解。

验收标准：

- 用户可选中文本添加备注。
- 备注显示在右侧面板。
- 备注可以编辑和删除。
- 备注可以作为知识卡片保存。

---

## 5. Epic：搜索问答

### US-008 混合搜索

作为用户，我希望输入关键词或自然语言问题后，系统同时搜索文档、实体、标注和图谱。

验收标准：

- 搜索请求支持关键词和自然语言。
- 返回本地文档命中。
- 返回相关实体。
- 返回相关标注。
- 返回图谱关系。
- 搜索结果包含命中片段。

### US-009 RAG 问答

作为用户，我希望系统根据知识库内容回答问题，并提供引用来源。

验收标准：

- 回答基于检索结果。
- 回答显示引用文档和片段。
- 没有足够证据时明确提示。
- 不允许出现无来源长答案。
- 支持点击引用跳转到原文。

---

## 6. Epic：实体和图谱

### US-010 自动识别实体

作为用户，我希望系统自动识别文档中的人物、公司、技术、股票和产业链实体，以便建立结构化知识。

验收标准：

- 文档解析后自动触发实体识别。
- 实体结果展示在文档详情中。
- 实体包含名称、类型、证据、置信度。
- 股票实体包含 ticker 和 exchange。
- 低置信实体进入待确认列表。

### US-011 合并重复实体

作为用户，我希望合并重复实体，例如 NVIDIA、英伟达、NVDA，以便图谱更干净。

验收标准：

- 系统给出重复实体建议。
- 用户可以查看合并前信息。
- 用户确认后合并别名和关系。
- 合并操作可回滚。

### US-012 查看实体图谱

作为用户，我希望查看某个实体的一跳关系，以便理解知识之间的关联。

验收标准：

- 用户点击实体后进入图谱视图。
- 默认显示一跳关系。
- 节点按类型显示不同颜色。
- 点击关系边可以查看证据。
- 支持继续展开节点。

---

## 7. Epic：深度研究

### US-013 创建深度研究任务

作为用户，我希望输入一个研究问题后，系统自动搜索本地知识库和网络资料，并生成研究报告。

验收标准：

- 用户可以创建研究任务。
- 系统生成研究计划。
- 系统搜索本地资料。
- 可选网络搜索。
- 系统保存来源列表。
- 系统生成报告。
- 报告可导入知识库。

---

## 8. Epic：NotebookLM

### US-014 导出 NotebookLM 资料包

作为用户，我希望选择一个专题并导出资料包，以便上传到 NotebookLM 使用。

验收标准：

- 用户可以选择分类或文档集合。
- 可以选择导出内容：正文、实体表、时间线、问题列表。
- 系统生成 ZIP。
- ZIP 中包含 README。
- 系统记录导出历史。

---

## 9. 关闭标准

一条用户故事必须满足以下条件才能关闭：

1. 代码已合并。
2. 接口已联调。
3. 单元测试通过。
4. 至少一条正常路径测试通过。
5. 至少一条异常路径测试通过。
6. 前端空状态和错误状态可用。
7. 文档或接口说明已更新。
