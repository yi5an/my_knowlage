# 数据库设计说明书

> 项目：KnowPilot AI Agent 本地知识库工具  
> 用途：指导产品、前端、后端、算法、测试和运维团队协同研发  
> 版本：v0.1  
> 日期：2026-05-15


## 1. 设计原则

1. 关系数据库保存主数据。
2. 向量库保存 Chunk 向量和检索 Payload。
3. 图数据库作为图查询加速层，可由关系库重建。
4. 文件存储保存原始文件和导出文件。
5. AI 生成结果必须保存来源、模型、版本和置信度。
6. 删除优先软删除，避免破坏知识溯源。

## 2. 核心数据流

```text
document_file
  ↓
document
  ↓
document_version
  ↓
document_chunk
  ↓
entity_mention
  ↓
entity
  ↓
entity_relation
  ↓

```

## 4. 主要表说明

### 4.1 document

用途：保存文档主记录。

关键字段：

| 字段 | 说明 |
|---|---|
| id | 文档 ID |
| workspace_id | 工作空间 |
| title | 标题 |
| source_type | file/url/manual/research |
| source_uri | 来源 URL 或路径 |
| file_id | 原始文件 ID |
| category_id | 主分类 |
| summary | 用户摘要 |
| ai_summary | AI 摘要 |
| parse_status | 解析状态 |
| index_status | 索引状态 |
| entity_status | 实体识别状态 |
| relation_status | 关系抽取状态 |
| sensitive_level | normal/private/secret |
| metadata | 扩展元数据 |

### 4.2 document_version

用途：保存文档正文版本。

业务规则：

1. 每次保存创建新版本。
2. 当前版本可在 document.metadata.current_version_id 中保存。
3. 回滚本质是创建新版本，内容来自历史版本。

### 4.3 document_chunk

用途：保存切分后的文档片段。

关键点：

1. chunk 与 version_id 绑定。
2. chunk 需要保存 page_no、heading、offset。
3. vector_id 对应向量库中的点 ID。
4. 文档版本变化后，应重新生成 chunk。

### 4.4 entity_type

用途：保存实体类型，包括系统内置、用户自定义、AI 建议。

字段：

| 字段 | 说明 |
|---|---|
| name | 实体类型名称 |
| domain | 领域，如技术、投资研究 |
| examples | 示例实体 |
| aliases | 类型别名 |
| rules | 识别规则 |
| source | system/user/ai_suggested |
| status | active/pending/rejected |
| confidence | AI 建议置信度 |

### 4.5 entity

用途：保存标准化后的实体。

业务规则：

1. normalized_name 用于查重。
2. aliases 保存别名。
3. properties 保存领域属性，例如股票 ticker。
4. verified 表示用户确认。
5. 合并实体时保留 source_entity_ids 到 metadata。

### 4.6 entity_mention

用途：保存实体在文档中的出现位置。

业务价值：

1. 支持来源回溯。
2. 支持实体出现次数统计。
3. 支持点击实体跳转原文。

### 4.7 entity_relation

用途：保存实体之间的关系。

业务规则：

1. 关系必须有 source_entity_id 和 target_entity_id。
2. 关系必须有 evidence_text。
3. 低置信关系进入待确认。
4. 关系可以被用户确认或拒绝。

## 5. 股票扩展设计

关键字段：

| 字段 | 说明 |
|---|---|
| entity_id | 对应 entity |

## 9. 删除策略

MVP 建议所有核心表增加：

```sql
deleted_at TIMESTAMPTZ NULL
```

软删除规则：

1. 删除文档不删除原始文件，先标记 deleted_at。
2. 删除实体不物理删除关系，关系标记失效。
3. 删除标注可软删除。
4. 图数据库数据可通过关系库重新同步。

## 10. 图数据库同步

同步策略：

```text
关系库变更
  ↓
outbox_event
  ↓
graph_sync_worker
  ↓
KuzuDB / NebulaGraph
```

建议增加 outbox_event 表：

```sql
CREATE TABLE outbox_event (
  id VARCHAR(64) PRIMARY KEY,
  event_type VARCHAR(128) NOT NULL,
  aggregate_type VARCHAR(64) NOT NULL,
  aggregate_id VARCHAR(64) NOT NULL,
  payload JSONB NOT NULL,
  status VARCHAR(32) DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT now(),
  processed_at TIMESTAMPTZ
);
```

## 11. 数据生命周期

| 数据 | 生命周期 |
|---|---|
| 原始文件 | 长期保存，除非用户彻底删除 |
| 解析文档 | 随版本长期保存 |
| Chunk | 当前版本必须保留，历史版本可配置清理 |
| 向量 | 可重建，允许删除后重建 |
| 图谱 | 可重建，以关系库为准 |
| 模型调用日志 | 默认保存 30-180 天，可配置 |
| 研究来源 | 随研究报告长期保存 |
