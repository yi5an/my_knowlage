# MVP 功能范围与版本计划

> 项目：KnowPilot AI Agent 本地知识库工具  
> 用途：指导产品、前端、后端、算法、测试和运维团队协同研发  
> 版本：v0.1  
> 日期：2026-05-15


## 1. MVP 目标

MVP 的目标不是一次性做完整 AI 知识操作系统，而是先打通最核心闭环：

```text
资料导入 -> 文档解析 -> 文档管理 -> 搜索问答 -> 实体识别 -> 图谱展示 -> 人工修正
```

MVP 要证明三件事：

1. 用户可以方便地把多种资料放进知识库。
2. 系统可以把资料变成可搜索、可问答、可标注的知识。
3. 系统可以抽取实体和关系，并用图谱方式辅助理解。

## 2. MVP 必做范围 P0

| 模块 | MVP 是否做 | 说明 |
|---|---:|---|
| 工作空间 | 做 | 单用户单工作空间起步，后续支持多工作空间 |
| 文件导入 | 做 | 支持 PDF、docx、xlsx、md、txt、图片 |
| URL 导入 | 做 | 支持网页正文抓取，失败时保存 URL 记录 |
| 文档解析 | 做 | 提取正文、表格、基础结构，保存为 Markdown |
| OCR | 做 | 图片 OCR、扫描 PDF OCR 基础能力 |
| 文档列表 | 做 | 分类、标签、状态、搜索 |
| 文档编辑 | 做 | Markdown 编辑优先，富文本后置 |
| 文档版本 | 做 | 每次保存生成版本，支持查看历史 |
| 标注备注 | 做 | 高亮、备注、摘录，先支持解析文档内容 |
| 全文搜索 | 做 | PostgreSQL FTS / SQLite FTS 起步 |
| 向量检索 | 做 | Qdrant 或 pgvector |
| RAG 问答 | 做 | 必须带引用来源 |
| 实体识别 | 做 | 通用实体 + 技术实体 + 股票/产业链基础实体 |
| 关系抽取 | 做 | 基于 Chunk 内实体共现和 LLM 判断 |
| 图谱展示 | 做 | 一跳展开、节点详情、证据查看 |
| 实体管理 | 做 | 编辑、合并、删除、确认 |
| 模型配置 | 做 | 支持本地模型和云模型配置 |

## 3. MVP 延后范围 P1/P2

| 模块 | 优先级 | 说明 |
|---|---:|---|
| 深度研究完整版 | P1 | MVP 先做简化版研究报告，后续做多步骤 Agent |
| NotebookLM 对接 | P1 | MVP 只做资料包导出，不做企业 API 自动同步 |
| 移动端 | P2 | MVP 不做 App，仅预留采集接口 |
| 多人协作 | P2 | MVP 不做权限体系复杂化 |
| 自动播客/测验 | P2 | 后续学习增强功能 |
| 浏览器插件 | P1 | 可作为第二阶段采集增强 |
| 股票实时行情 | P1 | MVP 只识别股票实体，不承诺实时行情 |
| 复杂产业链图谱 | P1 | MVP 先做实体关系，后续做专业产业链视图 |

## 4. 版本计划

### V0.1 基础知识库

目标：完成资料入库和文档管理。

- 项目初始化
- 文件上传
- URL 导入
- 文档解析
- 分类标签
- 文档列表
- 文档详情
- 文档编辑
- 基础全文搜索

### V0.2 RAG 问答

目标：支持语义搜索和带引用问答。

- Chunk 切分
- Embedding 生成
- 向量库写入
- 混合搜索
- Reranker
- RAG 问答
- 引用来源展示

### V0.3 实体和图谱

目标：形成知识图谱雏形。

- entity_type / entity / relation 表
- 实体识别
- 关系抽取
- 实体合并
- 图谱展示
- 节点一跳展开

### V0.4 深度研究简化版

目标：完成“问题 -> 搜索 -> 报告 -> 入库”闭环。

- 本地资料搜索
- 网络搜索
- 来源保存
- 报告生成
- 研究结果入库

### V0.5 NotebookLM 与生态增强

目标：支持资料包导出和外部工具协同。

- NotebookLM 资料包导出
- 浏览器剪藏
- 数据备份恢复
- 模型路由增强

## 5. MVP 成功标准

| 指标 | 目标 |
|---|---:|
| 常见文件导入成功率 | >= 90% |
| 文档解析可读率 | >= 85% |
| 搜索首屏命中率 | >= 80% |
| RAG 回答引用覆盖率 | >= 95% |
| 实体识别可用率 | >= 75% |
| 关系抽取可用率 | >= 60% |
| 图谱一跳展开响应 | <= 2 秒 |
| 用户完成一次资料沉淀闭环 | <= 5 分钟 |

## 6. 明确不做

MVP 不做以下能力：

1. 不做完整移动端 App。
2. 不做多人实时协作。
3. 不做股票交易、投资建议或自动下单。
4. 不承诺实时行情准确性。
5. 不做无证据的 AI 结论沉淀。
6. 不做完全自动不可控的实体合并。
7. 不把 NotebookLM 作为主数据库。
