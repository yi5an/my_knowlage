# 部署与运维手册

> 项目：KnowPilot AI Agent 本地知识库工具  
> 用途：指导产品、前端、后端、算法、测试和运维团队协同研发  
> 版本：v0.1  
> 日期：2026-05-15


## 1. 部署模式

KnowPilot 支持三种部署模式：

| 模式 | 说明 |
|---|---|
| 本地轻量版 | SQLite + 本地文件 + Ollama，适合个人 |
| Docker Compose 专业版 | PostgreSQL + Qdrant + Redis + MinIO + 图数据库 |
| 服务端团队版 | 后端服务集中部署，多用户访问 |

MVP 推荐先实现 Docker Compose 专业版，同时保留 SQLite 轻量版兼容。

---

## 2. 本地开发环境

### 2.1 基础依赖

- Python 3.11/3.12
- Node.js 20+
- Docker / Docker Compose
- PostgreSQL 16 可选
- Redis 7 可选
- Ollama 可选

### 2.2 后端启动

```bash
git clone <repo>
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 2.3 前端启动

```bash
cd frontend
npm install
npm run dev
```

## 3. Docker Compose 专业版

### 3.1 服务组成

```text
frontend
backend
worker
postgres
redis
qdrant
minio
nebula/kuzu
ollama
```

### 3.2 示例 compose

```yaml
services:
  backend:
    image: knowpilot-backend:latest
    ports:
      - "8000:8000"
    env_file:
      - .env
    depends_on:
      - postgres
      - redis
      - qdrant
      - minio

  worker:
    image: knowpilot-backend:latest
    command: celery -A app.workers.celery_app worker -l info
    env_file:
      - .env
    depends_on:
      - backend
      - redis

  frontend:
    image: knowpilot-frontend:latest
    ports:
      - "8080:80"

  postgres:
    image: postgres:16
    environment:
      POSTGRES_DB: knowpilot
      POSTGRES_USER: knowpilot
      POSTGRES_PASSWORD: knowpilot
    volumes:
      - ./data/postgres:/var/lib/postgresql/data

  redis:
    image: redis:7
    volumes:
      - ./data/redis:/data

  qdrant:
    image: qdrant/qdrant:latest
    ports:
      - "6333:6333"
    volumes:
      - ./data/qdrant:/qdrant/storage

  minio:
    image: minio/minio:latest
    command: server /data --console-address ":9001"
    ports:
      - "9000:9000"
      - "9001:9001"
    environment:
      MINIO_ROOT_USER: knowpilot
      MINIO_ROOT_PASSWORD: knowpilot_secret
    volumes:
      - ./data/minio:/data

  ollama:
    image: ollama/ollama:latest
    ports:
      - "11434:11434"
    volumes:
      - ./data/ollama:/root/.ollama
```

启动：

```bash
docker compose up -d
```

## 5. 初始化步骤

### 5.1 数据库迁移

```bash
docker compose exec backend alembic upgrade head
```

### 5.2 创建 MinIO Bucket

```bash
docker compose exec minio mc alias set local http://localhost:9000 knowpilot knowpilot_secret
docker compose exec minio mc mb local/knowpilot
```

### 5.3 拉取本地模型

```bash
docker compose exec ollama ollama pull qwen
docker compose exec ollama ollama pull bge-m3
```

模型名称以实际部署模型为准。

## 7. 备份与恢复

### 7.1 PostgreSQL 备份

```bash
docker compose exec postgres pg_dump -U knowpilot knowpilot > backup.sql
```

恢复：

```bash
cat backup.sql | docker compose exec -T postgres psql -U knowpilot knowpilot
```

### 7.2 MinIO 文件备份

直接备份：

```bash
tar -czf minio_backup.tar.gz ./data/minio
```

### 7.3 Qdrant 备份

MVP 可通过重新生成向量恢复。正式版应使用 Qdrant snapshot。

### 7.4 图数据库备份

图数据库可由关系库重建，但正式版仍建议定期快照。

## 8. 常见问题排查

### 8.1 文件上传成功但解析失败

检查：

1. worker 是否运行。
2. task_job 错误信息。
3. 文件格式是否支持。
4. OCR 服务是否可用。

### 8.2 搜索没有结果

检查：

1. 文档是否完成 chunk。
2. Embedding 是否成功。
3. Qdrant collection 是否存在。
4. 全文索引是否写入。

### 8.3 RAG 回答没有引用

检查：

1. 检索结果是否为空。
2. prompt 是否要求 citations。
3. chunk 是否有 doc_id 和 chunk_id。
4. LLM 输出是否通过 Schema。

### 8.4 图谱没有节点

检查：

1. 实体识别是否完成。
2. 关系抽取是否完成。
3. graph_sync_worker 是否运行。
4. 图数据库连接是否正常。

### 8.5 MinIO 无法访问

检查：

1. 后端和 MinIO 是否在同一 Docker 网络。
2. endpoint 是否使用容器名 `minio:9000`。
3. bucket 是否存在。
4. access key 是否正确。

## 9. 监控指标

| 指标 | 说明 |
|---|---|
| import_success_rate | 导入成功率 |
| parse_failed_count | 解析失败数 |

## 10. 发布流程

1. 合并 develop 到 release 分支。
2. 执行数据库迁移测试。
3. 执行回归测试。
4. 构建镜像。
5. 启动测试环境。
6. 验证导入、搜索、问答、图谱主链路。
7. 打 tag。
8. 发布正式环境。
9. 保留回滚镜像。

---

## 11. 安全建议

1. API Key 不写入代码。
2. 私密文档默认本地模型处理。
3. 云模型调用前记录用户设置。
4. 数据目录需要定期备份。
5. NotebookLM 导出需要用户确认。
6. 生产环境不要暴露数据库端口。
