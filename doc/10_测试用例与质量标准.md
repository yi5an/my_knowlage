# 测试用例与质量标准

> 项目：KnowPilot AI Agent 本地知识库工具  
> 用途：指导产品、前端、后端、算法、测试和运维团队协同研发  
> 版本：v0.1  
> 日期：2026-05-15


## 1. 测试目标

确保 KnowPilot MVP 在文档导入、解析、编辑、搜索、实体识别、关系抽取、图谱展示和 AI 问答方面达到可用质量。

---

## 2. 测试类型

| 类型 | 说明 |
|---|---|
| 单元测试 | 服务函数、解析器、工具函数 |
| 接口测试 | API 请求、响应、错误码 |
| 集成测试 | 导入到问答完整链路 |
| 前端测试 | 页面交互、状态展示 |
| AI 质量测试 | 实体、关系、摘要、问答质量 |
| 性能测试 | 搜索、图谱、导入耗时 |
| 回归测试 | 发布前全流程验证 |

---

## 3. 测试数据集

| 数据集 | 用途 |
|---|---|
| 技术文档样本 | 测技术实体、配置项、错误信息 |
| 投资研报样本 | 测股票、财务指标、产业链 |
| PDF 文档 | 测 PDF 文本提取 |
| 扫描 PDF | 测 OCR |
| Word 文档 | 测标题、表格、段落 |
| Excel 表格 | 测 Sheet 和表格解析 |
| 网页样本 | 测 URL 导入 |
| 图片截图 | 测图片 OCR |
| 错误文件 | 测失败处理 |

---

## 4. 文件导入测试

### TC-001 上传正常 PDF

步骤：

1. 上传普通 PDF。
2. 等待任务完成。
3. 打开文档详情。

预期：

- 任务状态为 completed。
- 生成文档记录。
- 正文可读。
- 可以搜索标题和正文关键词。

### TC-002 上传扫描 PDF

预期：

- 触发 OCR。
- OCR 结果保存为正文。
- OCR 失败时显示错误原因。

### TC-003 上传重复文件

预期：

- 系统识别 sha256 重复。
- 提示文件已存在。
- 用户可选择仍然导入或打开已有文档。

### TC-004 上传不支持格式

预期：

- 返回 UNSUPPORTED_FILE_TYPE。
- 前端显示错误提示。

## 5. 文档编辑测试

### TC-010 保存文档新版本

预期：

- 保存成功。
- version_no +1。
- 文档更新时间变化。
- 触发重新索引任务。

### TC-011 回滚历史版本

预期：

- 用户可以选择历史版本。
- 回滚后创建新版本。
- 原历史版本不被覆盖。

---

## 6. 标注测试

### TC-020 创建高亮

预期：

- 高亮保存到 annotation 表。
- 刷新页面后仍显示。
- 高亮可删除。

### TC-021 添加备注

预期：

- 备注绑定选中文本。
- 备注显示在右侧面板。
- 备注可编辑。

---

## 7. 搜索测试

### TC-030 关键词搜索

预期：

- 搜索关键词返回相关文档。
- 显示命中片段。

### TC-031 语义搜索

预期：

- 用户输入同义问题也能命中相关文档。
- 返回结果包含 chunk_id。

### TC-032 混合搜索

预期：

- 返回本地文档、实体和标注。
- 分数排序合理。

---

## 8. RAG 问答测试

### TC-040 有证据问答

预期：

- 回答包含引用。
- 引用能跳转到原文。
- 回答内容与原文一致。

### TC-041 无证据问答

预期：

- 不胡编。
- 明确提示知识库中证据不足。

### TC-042 股票类问答

预期：

- 回答提示资料日期。
- 不给出交易建议。
- 引用本地资料或网络来源。

---

## 9. 实体识别测试

### TC-050 技术实体识别

样本文本：包含 MinIO、FastAPI、Docker、ConnectTimeoutError、9000 端口。

预期：

- 识别产品、技术、错误信息、端口。
- 每个实体有 evidence。

### TC-051 股票实体识别

样本文本：包含“英伟达 NVDA 在 NASDAQ 上市”。

预期：

- 识别英伟达。
- ticker=NVDA。
- exchange=NASDAQ。

### TC-052 产业链识别

样本文本：包含 AI 算力上游芯片设计、中游服务器、下游云厂商。

预期：

- 识别产业链。
- 识别上游、中游、下游。

---

## 10. 关系抽取测试

### TC-060 明确关系

文本：“英伟达的高端 GPU 主要由台积电代工。”

预期：

- 抽取 英伟达 -> 依赖/供应关系 -> 台积电。
- 有 evidence。
- confidence >= 0.8。

### TC-061 不确定关系

文本中两个实体共现但没有明确关系。

预期：

- 不输出关系，或 confidence 低并进入待确认。

---

## 11. 图谱测试

### TC-070 一跳展开

预期：

- 点击实体显示一跳节点。
- 节点数量不超过限制。
- 点击边显示证据。

### TC-071 节点筛选

预期：

- 按类型筛选后画布刷新。
- 不相关节点隐藏。

---

## 12. 深度研究测试

### TC-080 创建研究任务

预期：

- 任务创建成功。
- 返回 job_id。
- 可查看进度。

### TC-081 研究结果入库

- 报告生成 document。
- 来源写入 research_source。

## 13. 性能标准

| 场景 | 目标 |
|---|---:|
| 文档列表加载 | <= 1 秒 |
| 关键词搜索 | <= 1 秒 |
| 语义搜索 | <= 2 秒 |
| RAG 问答首字返回 | <= 5 秒 |
| 图谱一跳展开 | <= 2 秒 |
| 10MB PDF 解析 | <= 60 秒 |
| 100 页 PDF 基础解析 | <= 5 分钟 |

---

## 14. AI 质量标准

| 项目 | 目标 |
|---|---:|
| 摘要可读率 | >= 85% |
| 实体识别可用率 | >= 75% |
| 关系抽取可用率 | >= 60% |
| RAG 引用覆盖率 | >= 95% |
| Schema 解析成功率 | >= 95% |

---

## 15. 发布准入标准

1. P0 用例全部通过。
2. P1 用例通过率 >= 90%。
3. 无阻塞级 Bug。
4. 文档导入主链路可用。
5. RAG 问答不出现明显无引用胡编。
6. 部署文档可按步骤成功启动。
