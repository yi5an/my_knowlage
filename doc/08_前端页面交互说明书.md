# 前端页面交互说明书

> 项目：KnowPilot AI Agent 本地知识库工具  
> 用途：指导产品、前端、后端、算法、测试和运维团队协同研发  
> 版本：v0.1  
> 日期：2026-05-15


## 1. 前端设计原则

1. 首页只做工作台，不堆叠完整功能。
2. 复杂功能放入独立页面。
3. AI 结果必须显示来源、置信度和可编辑入口。
4. 长任务必须显示进度。
5. 所有页面必须设计空状态、加载状态、失败状态。
6. 关键操作要可撤销或二次确认。

---

## 2. 全局布局

### 2.1 左侧导航

导航项：

- 首页工作台
- 导入中心
- 知识库
- 阅读标注
- 知识图谱
- 智能搜索
- 深度研究
- 实体管理
- NotebookLM
- 设置

### 2.2 顶部搜索

顶部搜索框用于全局搜索，不承载复杂筛选。点击后进入智能搜索页。

### 2.3 通用状态

| 状态 | 展示 |
|---|---|
| loading | 骨架屏或进度条 |
| empty | 空状态插画 + 引导按钮 |
| error | 错误原因 + 重试 |
| partial | 部分成功，展示失败项 |

---

## 3. 首页工作台

### 3.1 页面目标

首页只解决三件事：

1. 用户可以继续上次工作。
2. 用户知道今天需要处理什么。
3. 用户能快速进入核心功能。

### 3.2 组件

- 继续上次工作
- 今天需要处理
- 快速开始
- 最近活动
- 知识库状态
- 功能入口地图

### 3.3 不应该出现

- 大面积图谱画布
- 完整导入任务队列
- 完整搜索结果
- 深度研究报告正文
- 实体详情完整表单

### 3.4 交互

| 操作 | 行为 |
|---|---|
| 点击继续编辑 | 跳转阅读器并打开对应文档 |
| 点击查看研究进度 | 跳转深度研究详情 |
| 点击导入资料 | 跳转导入中心 |
| 点击搜索知识库 | 跳转智能搜索 |
| 点击待确认实体 | 跳转实体管理页待确认筛选 |

---

## 4. 导入中心

### 4.1 页面目标

让用户快速导入资料，并看到处理进度和失败原因。

### 4.2 组件

- 文件拖拽上传区
- URL 输入区
- 导入选项
- 任务队列
- 失败任务重试

### 4.3 交互

| 操作 | 行为 |
|---|---|
| 拖入文件 | 显示待上传列表 |
| 点击开始导入 | 创建导入任务 |
| 点击任务 | 展开任务详情 |
| 点击重试 | 重新创建任务或复用原任务重跑 |
| 点击打开 | 跳转文档详情 |

### 4.4 空状态

如果没有导入任务，显示：

```text
还没有导入任何资料。拖拽文件或粘贴 URL 开始构建知识库。
```

## 7. 知识图谱页面

### 7.1 页面目标

展示实体关系网络，支持扩展和证据查看。

### 7.2 布局

```text
左侧：筛选器
中间：图谱画布
右侧：节点/关系详情
```

### 7.3 交互

| 操作 | 行为 |
|---|---|
| 点击节点 | 右侧显示节点详情 |
| 双击节点 | 展开一跳关系 |
| 点击边 | 显示关系详情和证据 |
| 搜索实体 | 定位节点 |
| 勾选类型 | 过滤节点 |
| 点击路径查询 | 打开路径查询弹窗 |

### 7.4 性能规则

1. 默认只加载核心子图。
2. 单次渲染不超过 300 节点。
3. 大图必须分页或聚类折叠。

## 13. 前端质量要求

1. 所有接口请求必须有 loading 和 error 状态。
2. 长任务必须可查看进度。
3. 所有 AI 结果要有“重新生成”和“人工修改”。
4. 所有删除操作要二次确认。
5. 所有列表要分页或虚拟滚动。
6. 图谱大数据量必须限制渲染节点数。
7. 表单字段要做前端校验。
8. 所有页面应适配 1366px 以上桌面宽度。
