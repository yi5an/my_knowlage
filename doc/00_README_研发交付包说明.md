## 2. 使用方式

- 产品经理：重点阅读 01、02、03、08。
- 后端研发：重点阅读 04、05、06、07、11、12。
- 前端研发：重点阅读 08、04、13。
- 算法/AI 工程师：重点阅读 06、07、10。
- 测试工程师：重点阅读 02、03、10。
- 运维/部署人员：重点阅读 11、12。

## 3. 当前边界

当前文档用于启动 MVP 开发，不代表最终完整商业版范围。深度研究、NotebookLM 企业 API、移动端、团队协作等能力在 MVP 中部分降级或后置。

## 4. 建议执行顺序

1. 先冻结 MVP 范围。
2. 再确认数据库表与接口。
3. 然后拆分研发任务。
4. 前后端并行开发。
5. 最后补充 AI Pipeline、Prompt、测试集和运维部署。
